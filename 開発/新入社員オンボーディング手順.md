# 新入社員オンボーディング手順

新しく加わったメンバーが早期に活躍できるよう、入社後30日間の流れを定義します。受け入れ側のメンターが本手順に沿って支援します。

## 期間別マイルストーン

| 期間 | ゴール | 主な活動 |
| --- | --- | --- |
| 初日 | 環境準備 | アカウント発行・デバイス受け取り・チーム紹介 |
| 1週目 | 開発着手 | 環境構築・小さなタスク対応 |
| 2週目 | 領域理解 | ドメイン把握・ペア作業 |
| 1ヶ月目 | 自走開始 | 初PRマージ・振り返り面談 |

## 受け入れ側の役割分担

| 役割 | 担当 | 主な責任 |
| --- | --- | --- |
| メンター | 先輩エンジニア | 技術的な伴走 |
| バディ | 同部署メンバー | 生活面のサポート |
| マネージャー | 上長 | 期待値のすり合わせ |

オンボーディングのゴールは、本人が自走できる状態になることです。チェックリストは人事システムで管理しています。
